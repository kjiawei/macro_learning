trunk目录结构说明

NOTE：

1. 项目组成员务必遵循此目录结构说明中指明的用途来组织和放置相关文件，不得随意混用。
2. 源码工程中的目录结构在产品的运行环境下严格保持一致。默认的应用程序路径是/usr/local/app

bin             执行文件目录         

conf            配置文件目录

data            运行时保存的数据（包括临时数据）目录。在系统分区结构上可能会将此挂载点挂载到一个独立的nand分区上

docs            从源码中构建文档时的文档目录以及诸如INSTALL，BUILD等说明文档

include         公共头文件，和一些依赖的第三方组件的头文件

lib             库目录，存放共享库的二进制文件以便编译时链接或运行时加载

log             应用程序中产生的日志文件的存放目录

resources       资源目录，包括图片、多语言资源文件等

src             源码目录。其下按模块划分子目录
